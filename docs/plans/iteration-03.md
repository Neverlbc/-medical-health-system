# Iteration 03（两周）

目标：完善端到端质量保障与发布上线流程。

## 范围
- 端到端测试：关键用户路径（登录 → 档案 → 监测 → 预约 → 提醒）
- 性能与安全：分页与查询优化、鉴权细化、敏感信息保护
- 部署上线：容器化、反向代理、环境变量与密钥管理、基础监控

## 任务清单
  - 查询优化：常用列表加索引与查询审计；热点数据缓存 [M]
  - 安全：JWT 黑名单/续期策略（可选）、审计日志扩展 [M]
- 前端
  - 统一错误页与空态优化，权限拦截增强（按角色隐藏菜单）[S]
- 质量与发布
  - E2E 测试：Cypress/Playwright 打通关键路径；CI 中跑最小用例集 [M]
  - 性能/负载：后端压测基线（JMeter/ghz），前端懒加载/拆包 [M]
  - 部署：完善 `docker-compose` 与 Nginx；环境变量清单；一键启动脚本 [M]
  - 文档：部署与运维 FAQ、迁移指引、变更记录 [S]

## 验收标准
- 端到端关键路径可在 CI 中执行通过
- 在演示环境部署成功，主要接口 RT 与错误率达标

## 风险与缓解
- 云端密钥与配置：引入 `.env`/环境变量清单与示例，避免硬编码
